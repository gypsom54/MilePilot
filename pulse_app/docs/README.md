## Development principles

1. **Optimise for delight, not speed.**
2. **Technology should disappear.** The experience should feel human.
3. **Never hardcode greetings or copy in widgets.** Use the Brain or Copy Bible.
4. **When unsure, choose the calmer, simpler, more elegant solution.**
5. **Every sprint updates documentation.**
