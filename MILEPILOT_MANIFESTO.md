# MilePilot Manifesto

This isn't technical.

It's the philosophy of the product.

Every AI agent should read this before changing anything.

---

## Why MilePilot Exists

MilePilot exists to help self-employed drivers run better businesses.

We are not building another mileage tracker.

We are building the business companion for self-employed drivers.

## Our Mission

Help drivers:

- Save Time.
- Make More Money.
- Understand Their Business.
- Reduce Stress.

Every feature should support one or more of these goals.

## Design Philosophy

The interface should feel:

- Calm.
- Confident.
- Premium.
- Simple.
- Professional.

Every screen should feel like it belongs in a banking app rather than a spreadsheet.

## Development Philosophy

Never build features because they are interesting.

Only build features that improve a driver's working day.

Ask before every feature:

> "What problem does this solve?"

## Product Principles

1. Automation first.
2. Simplicity second.
3. Power third.

If something can be automatic, make it automatic.

## Tone of Voice

- Friendly.
- Professional.
- Encouraging.

Never robotic.

Never overly technical.

## Long-Term Vision

MilePilot becomes the operating system for self-employed drivers.

Mileage is only the beginning.

## The Golden Rule

Every release should make drivers think:

> "I'm glad I have MilePilot."

Not:

> "I suppose that's useful."

## Development Order

When making changes to MilePilot, follow this order:

1. Never break existing functionality.
2. Improve design before adding features.
3. Keep the app fast.
4. Keep the interface simple.
5. Match the MilePilot design system.
6. Test before committing.
7. Explain changes clearly.
