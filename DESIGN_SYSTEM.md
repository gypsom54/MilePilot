# MilePilot Design System

## Overall Feel

Premium mobile SaaS.

Clean, dark, modern and trustworthy.

The app should feel more like a banking or productivity app than a basic mileage tracker.

## Colours

Primary background:
Deep navy / near black

Primary accent:
Bright MilePilot blue

Text:
White for main text

Secondary text:
Soft blue-grey

Cards:
Dark translucent navy with subtle borders

Buttons:
Blue gradient with soft glow

## Typography

Use bold, clean headings.

Headings should feel confident but not cramped.

Avoid too many font sizes on one screen.

## Spacing

Use generous spacing.

Avoid cramming elements together.

Each screen should have one clear purpose.

## Buttons

Buttons should be:

- Rounded
- Premium
- Clean
- Slightly smaller than the current prototype
- Clearly tappable

## Inputs

Inputs should be:

- Clean
- Modern
- Rounded
- Not oversized
- Aligned with the rest of the screen

## Cards

Cards should be used carefully.

Avoid large empty cards.

Do not wrap simple inputs in massive panels.

## Icons

Use clean line-style icons.

Avoid cheap or cartoon-like icons.

No unnecessary car graphics on the welcome screen.

## Welcome Screen Target

The welcome screen should include:

- MilePilot wordmark
- Headline: Your driving business. On auto pilot.
- Short supporting explanation
- Feature row or feature list
- Animated typing prompt
- First-name input
- Continue button
- Data privacy note

It must look premium, clean and intentional.
